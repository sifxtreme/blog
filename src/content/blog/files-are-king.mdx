Files are king. I never put much into Claude memories or ChatGPT memories. Almost everything I care about lives as a plain file on my computer: notes, runbooks, code, half-finished ideas. It's all grep-able, all mine, all in one place.

That setup has one catch. When I'm away from my desk, I still want to work against those files. My answer is Remote Control. I pull up a Claude Code session on my phone and dictate what I want. Claude runs on my Mac, against my real files, and I talk to it from wherever I am.

My computer is always plugged in, so the Mac is always there to answer. The one thing I wanted to guarantee: at least one FREE remote session, ready to go, at all times. I didn't want to spin up a fresh one from my phone every time I had an idea. I wanted to open my phone and find a clean session already waiting.

So I wrote a small launchd agent that keeps one on standby. The full recipe is [on GitHub](https://gist.github.com/sifxtreme/878a4d1d6652334baa5fd34979c97ce8). Here's how it works.

## A remote session is just a file

A Remote Control session is a local `claude` process bridged to your phone. When it bridges, it writes a `bridgeSessionId` into `~/.claude/sessions/<pid>.json`, and that id is the URL you open on the phone:

```
claude.ai/code/<bridgeSessionId>
```

So finding my live remote sessions is just reading those files and keeping the ones whose process is still alive.

## "Free" means fewer than 3 of MY prompts

I call a session FREE if it has fewer than three of my own prompts in it. The tricky part is the counting. In a transcript, tool results and injected meta entries are ALSO role `user`, so a naive count overcounts by about 5x. The manager only counts real `type:user` lines that aren't `isMeta` and aren't a `tool_result`, and it early-exits at three, so even a giant transcript is cheap to check.

```python title="pool_manager.py" {6,9,11-14}
def count_prompts(sid, cap=3):
    tx = transcript_path(sid)
    n = 0
    with open(tx, "r", errors="ignore") as f:
        for line in f:
            if '"type":"user"' not in line:       # cheap pre-filter
                continue
            d = json.loads(line)
            if d.get("type") != "user" or d.get("isMeta"):
                continue
            content = d.get("message", {}).get("content")
            if isinstance(content, list) and any(
                b.get("type") == "tool_result" for b in content if isinstance(b, dict)
            ):
                continue                          # tool result, not something I typed
            n += 1
            if n >= cap:                          # ">= 3" is all we need to know
                break
    return n
```

## Spawn one when none is free

Every 15 minutes the agent reaps dead shells, counts prompts on each live session, and if nothing is free, spawns a replacement. The spawn runs detached inside tmux with a login shell, so the new `claude` inherits my full PATH and node environment. A headless `claude --remote-control` still registers a bridge and shows up on the phone.

```python title="pool_manager.py"
def spawn():
    name = "claude-free-" + str(int(time.time()))
    subprocess.run(
        ["tmux", "new-session", "-d", "-s", name, "-c", WORKDIR,
         "zsh", "-lc", "claude --remote-control standby"],
        check=True,
    )
    return name
```

Two rules keep it safe. It NEVER kills a live session; it only spawns when zero are free. And the only thing it ever destroys is a leftover tmux shell whose `claude` already exited. Any session I start by hand counts as the standby too, so it won't pile up redundant ones.

## The result

I open my phone and there's a clean Claude session already bridged to my Mac. I dictate what I want, it runs against my actual files, and by the time I'm back at my desk the work is done or waiting for me.

Files are king. Now they're king from my pocket too.
